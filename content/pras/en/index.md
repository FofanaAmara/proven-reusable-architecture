# PRA Registry

Welcome to the **Proven Reusable Architectures (PRAs)** registry of Banque Nationale du Canada.

## What is a PRA?

A PRA is a production-validated architectural solution that can be reused in different contexts. Each PRA contains:

- **Context**: Problem solved and use cases
- **Architecture**: Diagrams and technical documentation
- **ADRs**: Documented architectural decisions
- **Proven-in-use**: Real production implementations
- **Code Examples**: Code samples and configurations

## Registry Organization

The registry is organized into three scope levels:

### Bank-Wide (Cross-cutting)
PRAs validated for the entire bank (3+ multi-domain proven-in-use).

### Domain-Wide (Sectorial)
PRAs specific to a business domain:
- **Particuliers (Retail)**: Solutions for retail banking
- **Entreprises (Corporate)**: Solutions for SMEs and large corporations
- **Gestion de Patrimoine (Wealth Management)**: Solutions for high-net-worth clients

### Categories

- **Business**: Business processes, workflows, strategy
- **Application**: Frontend, backend, mobile applications
- **Data**: Databases, data warehouses, analytics
- **Technology**: Infrastructure, deployment, observability
- **Security**: Authentication, authorization, encryption
- **Integration**: APIs, messaging, data sync

## How to Use This Registry?

1. **Browse**: Use the [catalogue](/en/catalogue) to search and filter PRAs
2. **Explore**: Navigate the sections below by scope and category
3. **Implement**: Follow documentation and code examples
4. **Contribute**: Share your feedback and new PRAs

## PRA Statuses

- **Candidate**: 1+ proven-in-use, under validation
- **Approved**: 3+ proven-in-use, validated by governance
- **Deprecated**: Obsolete, do not use for new projects
